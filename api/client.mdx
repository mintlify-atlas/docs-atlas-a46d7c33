The `Client` class is the primary interface for all XMTP SDK functionality. It manages API connections, handles authentication, and provides access to conversations and preferences.

## Overview

The Client handles:
- Creating and managing XMTP identities
- Connecting to the XMTP network
- Managing conversations (Groups and DMs)
- Database encryption and storage
- Device synchronization

## Creating a Client

### create

Creates a new XMTP client with a signing key.

```swift
static func create(
    account: SigningKey,
    options: ClientOptions
) async throws -> Client
```

<ParamField path="account" type="SigningKey" required>
  The signing key used to authenticate and sign messages
</ParamField>

<ParamField path="options" type="ClientOptions" required>
  Configuration options for the client
</ParamField>

<ResponseField name="Client" type="Client">
  Returns a configured and authenticated Client instance
</ResponseField>

**Example:**

```swift
let options = ClientOptions(
    api: ClientOptions.Api(env: .production),
    dbEncryptionKey: encryptionKey
)
let client = try await Client.create(
    account: signingKey,
    options: options
)
```

### build

Builds a client from an existing public identity without requiring a signing key.

```swift
static func build(
    publicIdentity: PublicIdentity,
    options: ClientOptions,
    inboxId: InboxId? = nil
) async throws -> Client
```

<ParamField path="publicIdentity" type="PublicIdentity" required>
  The public identity to build the client with
</ParamField>

<ParamField path="options" type="ClientOptions" required>
  Configuration options for the client
</ParamField>

<ParamField path="inboxId" type="InboxId">
  Optional inbox ID. If not provided, will be fetched from the network
</ParamField>

<ResponseField name="Client" type="Client">
  Returns a Client instance without authentication capabilities
</ResponseField>

## Static Methods

### canMessage

Check if account identities can receive XMTP messages.

```swift
static func canMessage(
    accountIdentities: [PublicIdentity],
    api: ClientOptions.Api
) async throws -> [String: Bool]
```

<ParamField path="accountIdentities" type="[PublicIdentity]" required>
  Array of public identities to check
</ParamField>

<ParamField path="api" type="ClientOptions.Api" required>
  API configuration
</ParamField>

<ResponseField name="result" type="[String: Bool]">
  Dictionary mapping identity identifiers to their message capability
</ResponseField>

**Example:**

```swift
let api = ClientOptions.Api(env: .production)
let canMessage = try await Client.canMessage(
    accountIdentities: [identity1, identity2],
    api: api
)
```
