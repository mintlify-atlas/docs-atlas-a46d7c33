---
title: Dm
description: Direct messaging between two participants
---

The `Dm` struct represents a direct message conversation between two participants in XMTP. DMs are simpler than groups and optimized for one-on-one communication.

## Overview

DMs are created via the `Conversations` interface and support:
- One-on-one messaging
- Disappearing messages
- Message streaming
- Consent management

## Reading Messages

### messages

Get messages from the DM.

```swift
func messages(
    beforeNs: Int64? = nil,
    afterNs: Int64? = nil,
    limit: Int? = nil,
    direction: SortDirection? = .descending,
    deliveryStatus: MessageDeliveryStatus = .all,
    excludeContentTypes: [StandardContentType]? = nil,
    excludeSenderInboxIds: [String]? = nil,
    sortBy: MessageSortBy? = nil,
    insertedAfterNs: Int64? = nil,
    insertedBeforeNs: Int64? = nil
) async throws -> [DecodedMessage]
```

<ParamField path="beforeNs" type="Int64">
  Filter messages before this timestamp (nanoseconds)
</ParamField>

<ParamField path="afterNs" type="Int64">
  Filter messages after this timestamp (nanoseconds)
</ParamField>

<ParamField path="limit" type="Int">
  Maximum number of messages to return
</ParamField>

<ParamField path="direction" type="SortDirection">
  Sort direction: `.ascending` or `.descending` (default)
</ParamField>

<ParamField path="deliveryStatus" type="MessageDeliveryStatus">
  Filter by delivery status: `.all`, `.published`, `.unpublished`, `.failed`
</ParamField>

<ParamField path="excludeContentTypes" type="[StandardContentType]">
  Content types to exclude
</ParamField>

<ParamField path="excludeSenderInboxIds" type="[String]">
  Sender inbox IDs to exclude
</ParamField>

<ParamField path="sortBy" type="MessageSortBy">
  Sort field (sent time or inserted time)
</ParamField>

<ParamField path="insertedAfterNs" type="Int64">
  Filter by local insertion time
</ParamField>

<ParamField path="insertedBeforeNs" type="Int64">
  Filter by local insertion time
</ParamField>

<ResponseField name="messages" type="[DecodedMessage]">
  Array of decoded messages
</ResponseField>

<Note>
This returns raw messages without enriched metadata. For UI rendering, use `enrichedMessages()` instead.
</Note>

**Example:**

```swift
let messages = try await dm.messages(limit: 50)
for message in messages {
    print(message.content())
}
```

### enrichedMessages

Get messages with enriched metadata.

```swift
func enrichedMessages(
    beforeNs: Int64? = nil,
    afterNs: Int64? = nil,
    limit: Int? = nil,
    direction: SortDirection? = .descending,
    deliveryStatus: MessageDeliveryStatus = .all,
    excludeContentTypes: [StandardContentType]? = nil,
    excludeSenderInboxIds: [String]? = nil,
    sortBy: MessageSortBy? = nil,
    insertedAfterNs: Int64? = nil,
    insertedBeforeNs: Int64? = nil
) async throws -> [DecodedMessageV2]
```

Parameters are the same as `messages()` method.

<ResponseField name="messages" type="[DecodedMessageV2]">
  Array of messages with reactions, replies, and metadata "baked in"
</ResponseField>

<Tip>
Recommended for UI rendering. This method provides better performance and simpler code than fetching reactions and replies separately.
</Tip>
